# Changelog - v1.1

## Overview

v1.1 represents a major rewrite of fileserve with a shift from JavaScript to TypeScript, introduction of a configuration system, comprehensive security enhancements, and improved user experience with detailed directory indexes.

---

## TL;DR

- **Rewritten in TypeScript** for type safety and better IDE support
- **Added configuration system** (YSON-based) with sensible defaults
- **Major security improvements:** comprehensive deny list, path validation, XSS prevention
- **Interactive directory listings** with human-readable file sizes and relative timestamps
- **Express 4.18.1 → 5.2.1** (breaking changes possible)
- **Entry point moved:** `main.mjs` → `dist/main.js`
- **License added:** GPL-3.0-or-later (copyleft)
- **New dependencies:** @j0code/yson, temporal-spec, TypeScript, @types/* packages

---

## Breaking Changes

### Build Output Path
- **Changed:** Entry point moved from `main.mjs` to `dist/main.js`
- **Impact:** Deployment scripts and references to the entry point must be updated
- **Migration:** Update any deployment or invocation scripts to use `dist/main.js` instead of `main.mjs`

### License Change
- **Changed:** none → GPL-3.0-or-later
- **Impact:** All usage and distribution must comply with GPLv3 or later terms
- **Note:** This is a copyleft license requiring derived works to be open-source

### Module System
- **Added:** Explicit ES modules declaration via `"type": "module"`
- **Impact:** All code must use ES module syntax (import/export)
- **Breaking:** CommonJS require() is no longer supported

## New Features

### Configuration System
- **Added:** YSON-based configuration file support (`config.yson`)
- **Features:**
  - Load configuration from `config.yson` file in project root
  - Fallback to sensible defaults if config file is missing
  - CLI override: Pass `<path>` as first argument to override `basePath`
  - Configuration validation with helpful error messages
  - Support for: `basePath`, `port`, `exposeIndex`, `exposeHidden`, `denylist`, `allowlist`

### Security Access Control
- **Denylist:** Comprehensive default deny list including:
  - Linux/Unix system directories (`/etc`, `/sys`, `/proc`, `/boot`, etc.)
  - macOS system directories (`/Applications`, `/Library`, `/System`, etc.)
  - Windows system directories (`/Windows`, `/Program Files`, etc.)
  - Sensitive files (SSH keys, password hashes, `.env`, `.git`, etc.)
  - Docker-related paths
  - 100+ denied paths by default
- **Allowlist:** Optional allowlist for fine-grained access control
- **Hidden Files:** Toggle to expose/hide dot files via `exposeIndex` config

### Interactive Directory Indexes
- **HTML Generation:** Automatically generate beautiful directory index pages
- **File Metadata Display:**
  - File name with interactive links
  - MIME type
  - File size (human-readable format: KiB, MiB, GiB, etc.)
  - Modified time (relative durations, e.g., "2 days ago")
  - Changed time
  - Accessed time
  - Created time
- **Interactive Features:**
  - Breadcrumb navigation with links
  - Hover tooltips with absolute timestamps
  - Smart duration formatting (shows "just now", relative time, or absolute time)

### Temporal API Integration
- **Advanced Date/Time Handling:**
  - Uses ECMAScript Temporal API for robust date handling
  - Relative time display (e.g., "3 hours ago")
  - Intelligent unit selection based on time elapsed
  - Timezone-aware formatting
  - Support for computing durations across years, months, days, hours, minutes, seconds

### TypeScript Support
- **Full TypeScript Rewrite:**
  - All source code is now TypeScript with strict type checking
  - Typed configuration system with validation functions
- **Build System:**
  - `npm run build` compiles TypeScript to JavaScript
  - Source maps optional
  - Declaration files generated

## Improvements

### Code Quality
- **Type Safety:** All code is now fully typed with strict TypeScript compiler options
- **Security Validations:**
  - Path normalization to prevent directory traversal attacks
  - URL and HTML sanitization to prevent XSS attacks
  - Validation of all configuration values

### Server Features
- **Path Normalization Middleware:** Automatically normalizes and redirects malformed paths
- **Directory Handling:**
  - Automatic trailing slash redirect for directories
  - Configurable directory index exposure
  - Better error handling for unavailable directories
- **Error Handling:** Improved error messages for permission issues and missing files
- **Security Headers:** All responses include security headers:
  - `Access-Control-Allow-Origin: *` (CORS)
  - `Referrer-Policy: same-origin`
  - `Cache-Control: no-cache`
  - `X-Content-Type-Options: nosniff`

### User Experience
- **Human-Readable File Sizes:** Automatic conversion to appropriate units (B, KiB, MiB, GiB, TiB, etc.)
- **Smart Relative Timestamps:** "2 days ago" instead of absolute ISO timestamps
- **Breadcrumb Navigation:** Easy navigation through directory hierarchy
- **File Type Indicators:** CSS classes for different file types and MIME types
- **Project Information Footer:** Auto-generated footer with project name, version, author, and git commit link

### package.json Metadata
- **Added Comprehensive Fields:**
  - Keywords for discoverability
  - Homepage URL
  - Repository information
  - Bug tracker URL
  - License metadata
- **Type Definitions Export:** `"types": "./dist/main.d.ts"`

## Security Enhancements

### Path-Level Security
- **Directory Traversal Prevention:** Path normalization and validation prevents `../` attacks
- **Deny List Enforcement:** System directories and sensitive files are blocked by default
- **Allow List Support:** Optional whitelist restricts access to specific paths
- **Hidden File Control:** Dot files can be hidden via configuration

### XSS Attack Prevention
- **HTML Sanitization:** All user-controlled content is escaped
- **URL Encoding:** URLs are properly encoded to prevent malicious payloads
- **CSP-Ready:** Security header support for potential CSP implementation

### Information Disclosure Prevention
- **404 Pretend:** Denied paths return 404 instead of 403, preventing path enumeration
- **Hidden Metadata:** File metadata (size, timestamps) only shown for allowed paths

## Migration Guide

### For Existing Deployments
1. Update entry point reference from `main.mjs` to `dist/main.js`
2. Run `npm ci` to install new dependencies
3. Run `npm run build` to compile TypeScript
4. Create `config.yson` if needed (optional—defaults are sensible)

### For Developers
1. All code is now TypeScript - use `src/` for development
2. Run `npm run build` after making changes
3. New types available from `./dist/main.d.ts` for library usage
4. TypeScript strict mode enabled—fix any type errors

## Known Limitations

- Temporal API requires TypeScript polyfill definition until TS 6.0 (included in `temporal.d.ts`)
- Relative timestamp display requires browser support for `Intl.DurationFormat` (modern browsers only)
